# I18n API (Localization)

Translate your plugin for a global audience.

## Concept
Hardcoding English text (`"Welcome!"`) alienates players who speak other languages. Hytale's **I18n (Internationalization)** system allowing the *client* to translate text based on their settings.

## Workflow

### 1. Define Keys
Create a `lang/en_us.json` file in your assets.

```json
{
  "myplugin.welcome": "Welcome to the server, %s!",
  "myplugin.weapon.sword_name": "Doombringer",
  "myplugin.error.permission": "You do not have permission."
}
```

### 2. Send Localized Strings
Instead of sending a String, you send a `LocalizableString`.

```java
// The server sends the KEY ("myplugin.welcome") and the ARGS ("Steve").
// The CLIENT sees "myplugin.welcome" -> looks up translation -> inserts "Steve".
LocalizableString msg = new LocalizableString("myplugin.welcome", player.getName());

player.sendMessage(msg);
```

### 3. Using in Items/Entities
Most display names support localization.

```java
// The item name will update if the player changes their language!
ItemStack sword = new ItemStack(ItemType.DIAMOND_SWORD);
ItemMeta meta = sword.getMeta();

meta.setDisplayName(new LocalizableString("myplugin.weapon.sword_name"));
sword.setMeta(meta);
```

> [!TIP]
> **Fallback**: If a client doesn't have the translation key (e.g. strict German user), it will display the raw key or English fallback if configured. Always ship an `en_us.json`.
